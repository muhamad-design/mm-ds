AI & agents

The whole system ships as two markdown files an agent can read in one pass - one per theme, same token names.

Machine-readable specs#

design.md

Light theme. YAML frontmatter with every token value, then the rules in prose.

design.dark.md

Dark theme. Same token names, dark values.

How to use them#

Point an agent at either file and it has the entire system - no scraping, no screenshots. The two themes share names, so code can switch themes by swapping values only. Layout, shapes, motion, voice, and the do's and don'ts are documented there too.

Structure#

What each file contains

  • Frontmatter (YAML). colors (scales, alpha, P3 variants), typography, spacing, rounded, and components recipes.
  • Prose. Overview, color roles, layout rhythm, elevation, motion, shapes, component states, voice, and the do's and don'ts.
  • References. Component values point back at tokens with {colors.primary}-style paths.
