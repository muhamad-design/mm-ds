Getting started

mm-ds

A Geist-based design system for mm products: minimal, high-contrast, light and dark. Token-for-token aligned with Vercel's public Geist reference, and readable by people and AI agents alike.

Foundations#

Colors

Ten-step scales where every step has a job: backgrounds, borders, fills, text.

Typography

Geist Sans and Geist Mono across heading, label, copy, and button tokens.

Layout

A 4px spacing scale, a 1200px content column, and five breakpoints.

Elevation

Borders and tonal surfaces first; three quiet shadow tiers.

Motion

Instant by default; one springy curve when motion earns its place.

Shapes

Tight radii: 6, 12, 16, and full - one family per view.

Patterns#

Components

Buttons and inputs as token recipes, with sizes and states.

Voice & content

How interface copy is written: precise, verb-first, no filler.

Do's & don'ts

The rules that keep every surface consistent and accessible.

Principles#

What the system optimizes for

  • Minimal and high contrast. Generous whitespace, near-neutral surfaces, restrained color.
  • Color signals state. Accents mark links, errors, and warnings - never decoration.
  • Both themes are first-class. Every token name resolves in light and dark; designs never branch on mode.
  • Accessible by default. WCAG AA contrast and a visible focus ring everywhere.
  • Readable by machines. The same tokens ship as YAML, CSS variables, and per-theme markdown specs.

Consuming the system#

Web surfaces can link the generated custom properties directly; agents and tools read the markdown specs or the YAML token files.

/* web */
<link rel="stylesheet" href="assets/tokens.css">
color: var(--c-gray-1000);
background: var(--c-background-100);
border-radius: var(--rounded-sm);
